

1. Алгоритм для решения последующих задач
пишется новыми функциями в отдельных файлах в 
каталоге "algo4test". 

2.Варианты алгоритма пишутся внутри файла, имя которого == название алгоритма

3.В файле common модифицируется /создается новая  
версия функции 'compare_files' c  учетом алгоритма.

4. Тестируемый алгоритм и папка с данными указывается
в управляющем конфиг файле "config.py" в TASK, DATA_FOLDER.

Функция с решением типа "solution" импортируется в этом
же файле конфига. solution соответствует общему файлу алгоритма

Итоговый файл "mytest.py" не должен меняться. 
Запускается из терминала именно он.

В лог пишется результат тестирования алгоритма и его версий.
Файл лога внутри папки с данными конкретного алгоритма.
