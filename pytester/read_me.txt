

1. Алгоритм для решения последующих задач
пишется новыми функциями в отдельных файлах в 
каталоге "algo4test". 

2. Решение задачи алгоритма создается по примеру 
файла "strings.py" с подключением написанных функций и 
указанием в функции 'compare_files' внутри 
функции 'solution' соответствующего алгоритма анализа данных. 

3. Тестируемый алгоритм и папка с данными указывается
в управляющем конфиг файле "confif.py" в TASK, DATA_FOLDER.
Функция с решением типа "solution" импортируется в этом
же файле конфига. 

4. Итоговый файл "mytest.py" не должен меняться. 
Запускается из терминала именно он.
